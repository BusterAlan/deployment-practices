Prácticas de Despliegue en Google Cloud

Para equipos de 4 estudiantes - Aplicaciones Web en la Nube

Introducción

Estas tres prácticas están diseñadas para que equipos de 4 estudiantes aprendan a desplegar aplicaciones web en Google Cloud Platform. Cada práctica aumenta en complejidad y abarca diferentes servicios de GCP.

Organización de Equipos (4 personas)

Cada equipo debe asignar los siguientes roles para un trabajo eficiente:

  • Líder de Proyecto: Coordina las tareas, plazos y comunicación
  • Arquitecto Cloud: Diseña la infraestructura en GCP
  • Desarrollador Backend: Implementa la lógica de la aplicación
  • Desarrollador Frontend/DevOps: Desarrolla la interfaz y automatiza despliegues

Prácticas

Práctica 1: Sitio Estático con Cloud Storage

semana 5

Despliegue de un sitio web estático utilizando Google Cloud Storage y Cloud CDN.

Tecnologías: HTML, CSS, JavaScript, Google Cloud Storage, Cloud CDN

Pasos principales:

Crear un bucket en Cloud Storage
Configurar permisos de acceso público
Habilitar Cloud CDN para mejorar el rendimiento
Configurar un dominio personalizado (opcional)

Práctica 2: Aplicación con App Engine

semana 6

Despliegue de una aplicación web dinámica utilizando Google App Engine con base de datos Firestore.

Tecnologías: Python/Node.js, Google App Engine, Firestore, APIs

Pasos principales:

Desarrollar una aplicación web simple (lista de tareas o blog)
Configurar app.yaml para App Engine
Implementar base de datos Firestore
Desplegar en App Engine con gcloud CLI
Configurar dominio y SSL

Práctica 3: Aplicación con Kubernetes

semana 7-8

Despliegue de una aplicación contenerizada utilizando Google Kubernetes Engine (GKE) con CI/CD.

Tecnologías: Docker, Kubernetes, Google Kubernetes Engine, Cloud Build

Pasos principales:

Crear contenedores Docker para la aplicación
Configurar un cluster de GKE
Desplegar aplicación en Kubernetes
Configurar CI/CD con Cloud Build
Implementar monitoreo con Cloud Monitoring

Práctica 1: Sitio Estático con Cloud Storage (Detalle)

Semana 5 - Día 1-2

Configuración inicial de GCP: Crear proyecto, habilitar APIs necesarias, configurar facturación.

Semana 6 - Día 3-4

Desarrollo del sitio estático: Crear HTML, CSS y JavaScript para un sitio de portfolio personal.

Semana 7 - Día 1-2

Crear bucket en Cloud Storage y configurar para hospedaje web.

Semana 8 - Día 3-4

Subir archivos, configurar CDN y probar el sitio desplegado.

  • Documentación oficial: Hospedaje web en Cloud Storage
  • Tutorial: Crear tu primer sitio en GCS

Criterios de Evaluación - Práctica 1

  • Sitio web accesible públicamente (20%)
  • Configuración correcta de permisos (20%)

  • Calidad del código y diseño (20%)
  • Documentación del proceso (20%)

Preparar app.yaml y pruebas locales con el emulador.

Semana 3

Despliegue en App Engine y configuración de dominio personalizado.

Recursos para la Práctica 2

Criterios de Evaluación - Práctica 2

  • Aplicación desplegada y funcional (25%)
  • Operaciones CRUD con Firestore (25%)

Semana 2 - Día 1-2

Semana 2 - Día 3-4

Desplegar aplicación: Crear deployments, servicios y configmaps.

Semana 3

Implementar CI/CD: Configurar Cloud Build para automatizar despliegues.

Semana 4

Monitoreo y ajustes: Configurar Cloud Monitoring y optimizar recursos.

Recursos para la Práctica 3

Criterios de Evaluación - Práctica 3

  • Aplicación ejecutándose en GKE (25%)
  • Configuración de CI/CD con Cloud Build (25%)
  • Dockerización correcta de la aplicación (20%)
  • Configuración de monitoreo (15%)
  • Documentación técnica (15%)

Consejos para el Trabajo en Equipo

  • Utilizen GitHub para control de versiones y colaboración
  • Dividan las tareas según los roles asignados
  • Mantengan reuniones regulares (2-3 veces por semana)
  • Documenten todo el proceso y los problemas encontrados
  • Prueben cada componente individualmente antes de integrarlos
